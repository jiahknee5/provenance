Provenance · personalization router

The identity key you can prove — not a guess — gates what you may personalize on. The router is deterministic; the LLM only drafts; the Gate disposes.

Every visitor IP runs through one tree: classify the network from real ip-api flags → score per-field confidence → route to a personalization tier. A specific industry is asserted only at high confidence; everything else falls back to a true neutral default. The same tree decides whether a B2B visitor earns competitor-aware copy — and the Gate verifies every comparative claim before it ships.

Exhibit 2 — Industry needs a confidence gate — assertion is earned, not defaulted

ALLUDE HIGH

Clean corporate IP · PDL returns an industry · it maps to a known sector (optionally corroborated by the company domain).
Ship industry-tailored copy + backdrop. Still allude — shape to the sector, never recite that we detected it.

ALLUDE MEDIUM

Company resolved but industry is sparse/ambiguous, or a corporate-VPN netblock (not a clean office IP).
Shape tone only — region + company-aware framing, neutral backdrop. Hold the sector claim.

HOLD LOW / UNKNOWN

No company resolved (consumer ISP, mobile, VPN, bot) — nothing to attribute an industry to.
Never assert a sector. Fall to the General neutral default. ← the fix for "why is my industry mining?"

Why this matters: location is ~city-accurate on fixed-line broadband, so we treat it as generally correct (and still allude, never recite). Industry is a second-order inference (IP→company→PDL→sector) — each hop loses confidence, so it must clear a bar before it changes the page. A confident-looking sector on a guess is the failure mode we just removed.

Exhibit 3 — Where it lives in the agent graph: a deterministic router, LLM leaves, one Gate

det Classify
ip-api flags → network type (the 6-way fork above).

Per-field confidence: location / company / industry. Optional LLM corroboration only.
det Route
→ Tier 0 · 1 · 2 · 3. Pure, reproducible, itself provable.
llm Research
Tier 3 only — competitor brief: provable category differentiators.

Proposes headline/sub/CTA from the brief. Claude Haiku. Inert offline.
gate The Gate
Disposes. Verifies every claim vs source + surface policy. Comparative claims = highest scrutiny.

Backdrop / accent / tone from firmographic + confidence.
det Render
+ provenance receipt: every line's source, basis, policy.

↩ Fallback path: any line the Gate rejects → the deterministic angle template ships instead (angle_copy). The page never blocks; it degrades to what's provable. So yes — this tree is part of the agent graph: it's the router in front of the existing propose→dispose flow. Only two nodes are LLM (Research, Copy); the spine is deterministic, and the Gate stays the single point of disposition.

deterministic — provable, cheap, reproducible LLM agent — proposes only the Gate — disposes
Can ≠ should. Every signal here carries a say / allude / hold surface policy. Geo + firmographic are allude (involuntary — shape, don't recite). "Creepy mode" is this exact same data with the policy turned off — reciting the company, the precise location, or an inferred competitor. The Gate makes that arm provably never selected, not merely discouraged.

Grounded in pipeline/personalization/scene.py (reverse_ip, resolve_ip network classification, PDL enrich) and creative.py (copy agent + Gate). Tiers 0–1 ship today; the corporate-VPN check, confidence scoring, and Tier 3 competitor agent are the proposed additions. Real fields only — no fabricated provider claims.
