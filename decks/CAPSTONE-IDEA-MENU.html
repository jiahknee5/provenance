Build the trust layer for agents

Six differentiated, cutting-edge capstone candidates — each aimed at a specific, currently-unsolved AI problem people are arguing about on HN, X, and arXiv right now (June 2026). Ranked for ambition, AI-as-primitive, 2-week feasibility, recruit pull, and career authority.

The through-line. The hottest, best-sourced gap in mid-2026 isn't capability — it's trust: "verification is the new bottleneck" (Karpathy). Benchmarks get gamed, LLM-judges fail >50% of bias tests, RLVR models reward-hack, agents collapse on long horizons, and prompt injection is now contained, not solved. The brief's own Direction-B example — one shared agent, the team just verifies — sits exactly on this gap. The top picks below all build some form of calibrated trust / verification for AI output; that's the skill that reads as authority to any future employer.

The candidates — ranked

1

★ The Oversight Layer

One shared agent does the work; a learned model decides what a human must verify.

Dir. B · different agent design
The problem & why it's hard
The brief's headline question — could a team run through one shared agent and just verify it? — fails today because verification doesn't scale: a human babysitting a swarm goes careless, and agents can't tell you which of their outputs to distrust. The hard bet is building a calibrated risk model that triages a stream of agent actions into auto-approve / human-review / block — when the agent's own confidence is known to be miscalibrated.
AI as primitive
A trained risk/confidence model (on action features + diff blast-radius + hidden-state probe + judge-ensemble disagreement) is the product. Remove it and you're back to reading everything by hand.
2 weeks: MVP → ambitious
MVP: one agent does a real task stream; a trained gate routes only risky outputs to a human; measure caught-error rate vs. % reviewed. Ambitious: active-learning loop where human verdicts retrain the gate; an org-throughput sim showing 1 verifier safely covering N agents.
Recruit hook
Everyone is anxious about trusting agents. "We're building the layer that lets one person safely supervise ten agents" recruits itself.
Key risk
Needs a labeled "was this output actually wrong?" set — bootstrap from a task domain with cheap ground truth (code tests, structured tasks) so the gate has signal.
Authority signal
"I understand the real bottleneck of agentic orgs and built the calibrated oversight layer." Directly on the #1 industry conversation.

The Quality Oracle

A small model reads a big LLM's hidden states to predict — before you pay for it — whether the answer will be wrong, then routes/gates/escalates.

Dir. A · ML + LLM
The problem & why it's hard
Routers today use crude heuristics, and an LLM's self-reported confidence is poorly calibrated and collapses under distribution shift. The bet: a trained, calibrated predictor on internal activations beats self-verbalized confidence at lower cost — a real supervised-learning problem (AUC, calibration, ECE under shift) grafted onto a generative one.
AI as primitive
Classical ML (probe/regressor on hidden states) does what the LLM can't (know when it's wrong); the LLM does what ML can't (generate). Neither alone is the product.
2 weeks: MVP → ambitious
MVP: probe on one open model + one dataset, plot the cost-accuracy frontier vs. self-confidence and length baselines. Ambitious: generalize across domains, add an inline hallucination gate, cascade with learned accept heads.
Recruit hook
"We can look inside the model and predict its mistakes" is a magnet for the ML-leaning teammates.
Key risk
Activation extraction + a labeled correctness set is fiddly; pick an open-weights model and a domain with automatic correctness checks to stay in scope.
Authority signal
"I probe LLM internals and build calibrated predictors / routers." The densest pure-AI-engineering credential on this menu.

The Verifier Bake-Off

One question — "should we trust this output?" — attacked five ways, measured head-to-head to see what actually wins.

Dir. C · one problem, many ways
The problem & why it's hard
Take a single verification problem and pit self-consistency vs. LLM-judge vs. semantic-entropy vs. a trained hidden-state probe vs. a judge-ensemble on the same task, scoring cost, accuracy, and calibration. Hard because there's no agreed answer to "which verifier wins where," and building a fair, un-gameable harness is itself the Berkeley-RDI problem.
AI as primitive
You build the trained probe and the ensemble, not just call APIs — so it's also ML+LLM under the hood, with a comparative-eval spine.
2 weeks: MVP → ambitious
MVP: 3 methods on one dataset + a clean leaderboard. Ambitious: 5 methods across 2 domains, an adversarial split that exposes each method's failure mode, a "router that picks the cheapest verifier that's good enough."
Recruit hook
"We're going to settle which verifier actually works" — appealing to rigorous, eval-minded teammates; very low risk of not shipping.
Key risk
Lowest ambition ceiling of the top three — must avoid being "a benchmark." The adversarial split + the cost frontier is what makes it ambitious.
Authority signal
"I can rigorously evaluate AI systems and know which method wins where" — the eval literacy every serious AI team needs and few have.

The Tool/Think Bandit

A learned contextual-bandit policy decides, per step: answer directly, think, or call which tool — trained on reward = correctness − cost − latency.

Dir. A · ML + LLM
The problem & why it's hard
Agents waste tokens because the LLM decides its own tool use; "tool choice is the missing policy in AI agents." Hard part: sparse/delayed credit assignment and featurizing context so a cheap policy can beat always-think and self-decide on the cost-adjusted frontier.
AI as primitive
An RL/bandit policy steers the LLM at runtime — the LLM is the actuator, the policy is the brain deciding when to spend.
2 weeks: MVP → ambitious
MVP: LinUCB/Thompson over {answer, think, tool-k} on one task suite, vs. baselines. Ambitious: off-policy learning from logged traces, multi-turn credit assignment, transfer across task types.
Recruit hook
"We make agents 3× cheaper at the same accuracy" — a crisp, measurable win.
Key risk
Reward shaping and the eval suite are where it lives or dies; keep the action space small and the reward clean.
Authority signal
"I apply RL/bandits to control LLM systems" — strong, somewhat narrower than #1/#2.

The Self-Healing Agent

Execution-time plan verification + checkpoint/repair that catches long-horizon collapse before it cascades — with a learned collapse-predictor.

Dir. B/A · agent design + ML
The problem & why it's hard
Long-horizon agents collapse structurally (85%/step compounds to ~27%), and ~60% of failures are harness-level, not model-level. The bet: a monitor that predicts imminent collapse from trajectory features and triggers re-plan / rollback to a consistent checkpoint. Hard because "what is a sufficient checkpoint of inferred context/plans, and how to resume without inconsistency" is explicitly unsolved.
AI as primitive
A trained collapse-predictor + a verification model gate the agent's own execution — the loop self-corrects instead of barreling to failure.
2 weeks: MVP → ambitious
MVP: a checkpoint/replan wrapper on one long-horizon benchmark, end-to-end success vs. vanilla. Ambitious: learned collapse-predictor from the failure taxonomy; show recovery on tasks the baseline always loses.
Recruit hook
"We make agents that finish 50-step tasks instead of dying at step 12."
Key risk
Highest-risk of the set for two weeks — needs a working long-horizon harness first. Big ceiling if it lands.
Authority signal
"I make agentic systems reliable at long horizons" — squarely the production pain employers feel.

The Black Box Recorder

Deterministic record-and-replay + a grader sandbox: make agent failures reproducible, and catch agents red-handed gaming the test.

Dir. C/B · systems + eval
The problem & why it's hard
Same input → different tool sequence even at temp 0, so failures aren't reproducible; and benchmarks get gamed because the agent shares the grader's environment. The bet: capture the full execution (model calls, tool I/O, retrieval) for deterministic replay, plus a sandbox that isolates the grader — then detect gaming behaviors automatically.
AI as primitive
Lighter on ML (more systems) unless you add a learned anomaly detector over traces that flags reward-hacking patterns — recommended to keep it "AI."
2 weeks: MVP → ambitious
MVP: record/replay one agent deterministically + reproduce a planted bug. Ambitious: a trace anomaly detector that catches grader-gaming (git-log peeking, monkey-patched scorers) on real trajectories.
Recruit hook
"We caught GPT-class agents cheating their own benchmarks" — a great demo moment.
Key risk
Can read as plumbing; the anomaly-detection + the "caught it cheating" demo is what gives it teeth and ambition.
Authority signal
"I build the eval/observability infra serious AI teams can't ship without."
