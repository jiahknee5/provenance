The Oversight Layer

One shared agent does the work. The team only verifies. We build the model that makes "only verify" actually scale.

Thesis. An agentic org's ceiling isn't how smart the agent is — it's how much of its output one human can safely trust. "Verification is the new bottleneck" (Karpathy). We train a calibrated risk model that triages every agent action into auto-approve / human-review / block, so one verifier can safely cover many agents. The unit of progress isn't model IQ — it's the oversight ratio.

Direction B · rethink agents ML + LLM under the hood 4 ownable surfaces Scored on ambition

The reframe (the spiky POV)

We're measuring agents wrong.

Every benchmark scores capability — can the agent do the task? But the brief's own question is the real one: could a company run through one shared agent, with the team just verifying it? That fails today not because agents are dumb, but because verification doesn't scale — a human babysitting a swarm goes careless, and the agent can't tell you which of its outputs to distrust (its self-reported confidence is miscalibrated). Capability is commoditizing. Trusted throughput is the bottleneck.

The oversight ratio — how many agents one verifier can safely cover at a fixed error budget ε — is the metric we optimize, and the thing nobody is building a model for.
1

The problem — and why it's technically hard

"Let one agent do the work and have humans verify" only works if verification is cheap and reliable. It is neither. Making it both is a stack of genuinely hard problems:

  • Calibration under distribution shift. The gate must output calibrated probabilities (low ECE), not just rank-order — a 0.8 must mean 0.8 — and stay calibrated as tasks drift. LLM self-verbalized confidence is documented to be poorly calibrated and to collapse under shift, so we can't just ask the agent.
  • No ground truth at decision time. We're predicting "is this output wrong?" before we know the answer. That's selective prediction (classification with a reject option), and its whole value is in the regime where you can't check — so the calibration learned where you can check has to transfer to where you can't.
  • The human is in the loop and degrades. Reviewer accuracy drops with volume (automation complacency). The triage must respect a human attention budget and route to maximize caught-errors at that budget — an oversight-economics optimization, not a plain classifier.
  • Cost-sensitive, not symmetric. A wrong typo and a wrong rm -rf are not the same error. The gate must weight predicted-error by blast radius (reversibility, money, prod, external comms) — the agent equivalent of the "lethal trifecta."
  • Scarce, expensive labels. Every human verdict is a costly label. To improve, the system must ask for labels where they're most informative (active learning) and retrain online without destabilizing calibration.

Ambition lives in turning "humans verify the agent" from a slogan into a measurable, calibrated control system — and reporting the oversight ratio it buys.

Project direction

Primary fit · Direction B

A philosophically different take on agents. We don't add another agent or a better prompt — we invert the org: the agent is the worker, the humans are a verification function, and the AI primitive is the model that allocates scarce human attention across a stream of agent work. It is the literal build-out of the brief's own example, made rigorous.

…with a real ML + LLM core (it qualifies for A too)

  • The gate is a trained, calibrated risk model that fuses classical-ML signals (a probe on the LLM's hidden states, OOD/novelty scores, a structured blast-radius model) with LLM-derived signals (judge-ensemble disagreement, semantic entropy). Classical ML does what the LLM can't — know when it's wrong — and the LLM does the work. Neither alone is the product.
  • Why not "one problem, many ways" (C)? We do compare signal sources, but the project's spine is the inverted org + the calibrated control loop, not a method bake-off. Lead with B.
3

AI is the primitive, not a bolt-on

The tell: delete the risk model and the whole premise collapses — "verify everything by hand" is just slower than doing the work yourself. The model that decides what a human must look at is the entire product. The human sets the error budget; the model spends their attention optimally.

┌─────────────── active-learning loop ─────────────────┌ │ │ agent output ─► SIGNAL EXTRACTORS ─► CALIBRATED RISK MODEL ─► TRIAGE POLICY ────────────────────────────────┦ stream • hidden-state probe p(correct), low ECE (risk-coverage ► AUTO-APPROVE ► ship (1 shared • judge-ensemble var. + E[cost × blast-radius] + attention budget) ► HUMAN REVIEW ► verdict ─┦ agent, • semantic entropy ► BLOCK / REPAIR ► agent │ N tasks) • blast-radius score │ • OOD / novelty scarce human verdicts ─── retrain (uncertainty sampling) ◄────────┘
  • Signal extractors. Per agent output, compute a feature vector: a probe on intermediate hidden states (open model), variance across a diverse-family LLM-judge ensemble, semantic entropy from N samples, an OOD/novelty score vs. the training distribution, and a structured blast-radius score (reversible? touches prod/money/external comms?).
  • Calibrated risk model. A small supervised model maps those features to a calibrated p(correct) (temperature/isotonic calibration; we report ECE + reliability diagrams, not just AUC) and an expected cost of error = (1 − p) × blast-radius.
  • Triage policy. Given the human's remaining attention budget, choose the lane that maximizes caught-error value — formally walking the risk-coverage curve: auto-approve the safe high-confidence tail, escalate the uncertain/high-blast-radius middle, block the predicted-dangerous.
  • Active-learning loop. Human verdicts are the labels. We request them where the model is most uncertain, retrain, and watch calibration improve as the team works — the gate gets sharper the more it's used.
4

Scope — what "working" means in two weeks

We anchor in one domain with automatically checkable ground truth (e.g., code-fix tasks with hidden tests, or structured extraction against a gold set) so we can train and honestly measure calibration — then probe whether it transfers to where you can't check.

Minimum shippable — must hit
  • One shared agent runs a real task stream in the ground-truth domain.
  • Risk model fusing 2–3 signals (judge-ensemble disagreement + semantic entropy + blast-radius) into a calibrated p(correct).
  • A triage UI: outputs sorted into auto-approve / review / block; the human reviews only what's flagged.
  • The headline chart — caught-error rate vs. % human-reviewed — beating self-confidence and random baselines, plus a reliability diagram (calibration).
Ambitious — scored on ambition
  • Hidden-state probe added (open-weights model) — the internals signal self-confidence throws away.
  • Active learning, live — human verdicts retrain the gate; ECE visibly drops over the session.
  • The oversight-ratio curve — "1 verifier safely covers N agents at error budget ε," the money chart.
  • Transfer probe — train where truth is checkable, test calibration in a domain where it isn't. This is the whole bet, stated honestly.
  • Cost-sensitive triage — blast-radius-weighted "block the irreversible" lane.

Why it's compelling — the recruit hook

  • It's the conversation everyone is having. Nobody trusts a swarm of agents. "We're building the layer that lets one person safely supervise ten agents" recruits itself — and it's the brief's own headline question, answered.
  • Spiky, defensible POV: "Capability is commoditizing; trusted throughput is the bottleneck. We don't measure model IQ — we measure the oversight ratio, and we move it." Memorable to judges and recruits alike.
  • It demos like a control room — watch a stream of agent work flow through the gate, watch the dangerous one get blocked with a reason, watch the review queue shrink as the gate learns. People want to be on stage for that.

Hard parts / risks — the bet and where it breaks

The bet: a trained, calibrated risk model lets one human safely vouch for far more agent output than reading it all — and the calibration learned where truth is checkable transfers to where it isn't.

RiskWhere it breaks · mitigation
Calibration is the whole game
(top risk)		If p(correct) isn't calibrated, the triage is arbitrary. Mitigation: temperature/isotonic calibration; report ECE + reliability diagrams, not just AUC; anchor in a ground-truth domain so calibration is measurable, not assumed.
Does it transfer off-ground-truth?The value is triaging where you can't check; if calibration doesn't transfer from the checkable domain, the claim weakens. Mitigation: make the transfer experiment a first-class deliverable; report transfer lift vs. nothing; be honest about the gap rather than hide it.
The human-degradation modelThe oversight-ratio curve depends on how reviewer accuracy decays with load — a modeling assumption judges will poke. Mitigation: ground it in automation-complacency literature + a small real human-in-the-loop trial; present as a sensitivity range, not a point.
The agent games the gateIf the agent learns the gate's signals, it can reward-hack them (the RLVR failure mode). Mitigation: keep some signals hidden from the agent; flag adversarial robustness as scoped future work, not a solved claim.
Label scarcity / cold-startLittle labeled "was this wrong?" data at the start. Mitigation: bootstrap from the auto-checkable domain; uncertainty-sampling active learning for the rest.
Hidden-state accessProbing internals needs open weights. Mitigation: MVP runs entirely on closed models via judge/entropy/blast-radius; the probe is an ambitious-tier add, not a dependency.
